## Option A — NPM (React / Next.js)

### 1. Install

```bash
npm install @zentrafi/terminal
```

<Info>
**Peer dependencies required:** `react >=18`, `react-dom >=18`, `wagmi ^2.19.5`, `viem ^2`, `@tanstack/react-query ^5`, `@rainbow-me/rainbowkit ^2`
</Info>

### 2. Import CSS

```tsx
// In your root layout or _app.tsx
import "@zentrafi/terminal/dist/styles.css"
```

### 3. Render

```tsx
import { ZentraTerminal } from "@zentrafi/terminal"

export default function MyPage() {
  return (
    <ZentraTerminal
      initProps={{
        displayMode: "Integrated",
        themeType: "Dark",
        walletConnectProjectId: "YOUR_WALLETCONNECT_PROJECT_ID",
      }}
    />
  )
}
```

That's it. The terminal ships with its own wallet connection (`independentWallet: true` by default) so no provider setup is needed on your end.

## Option B — HTML / CDN (no build step)

```html
<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="https://cdn.zentrafi.xyz/terminal/style.css" />
</head>
<body>
  <div id="swap-terminal"></div>

  <script src="https://cdn.zentrafi.xyz/terminal/main.js"></script>
  <script>
    window.ZentraX.init({
      containerId: "swap-terminal",
      displayMode: "Integrated",
      themeType: "Dark",
      walletConnectProjectId: "YOUR_WALLETCONNECT_PROJECT_ID",
    })
  </script>
</body>
</html>
```

### Auto-init via data attributes

If you prefer zero JavaScript, add a `data-zentra-terminal` attribute and the terminal initialises itself on `DOMContentLoaded`:

```html
<div
  data-zentra-terminal
  data-mode="Widget"
  data-theme="Dark"
  data-position="bottom-right"
  data-wallet-connect-id="YOUR_PROJECT_ID">
</div>

<script src="https://cdn.zentrafi.xyz/terminal/main.js"></script>
```

Supported data attributes:

| Attribute | Values |
|-----------|--------|
| `data-mode` | `Integrated` · `Modal` · `Widget` |
| `data-theme` | `Dark` · `Light` |
| `data-position` | `bottom-right` · `bottom-left` · `top-right` · `top-left` |
| `data-wallet-connect-id` | Your WalletConnect Project ID |

## Using your own wallet setup

If your dApp already has wagmi + RainbowKit configured, pass `independentWallet: false` and wrap the terminal inside your existing providers:

```tsx
import { ZentraTerminal, TerminalProviders } from "@zentrafi/terminal"
import "@zentrafi/terminal/dist/styles.css"

// Your existing wagmiConfig and queryClient here...

export default function SwapPage() {
  return (
    <WagmiProvider config={wagmiConfig}>
      <QueryClientProvider client={queryClient}>
        <RainbowKitProvider>
          <ZentraTerminal
            initProps={{
              displayMode: "Integrated",
              independentWallet: false,
            }}
          />
        </RainbowKitProvider>
      </QueryClientProvider>
    </WagmiProvider>
  )
}
```

<Warning>
When `independentWallet: false`, you must provide all three providers (`WagmiProvider`, `QueryClientProvider`, `RainbowKitProvider`) yourself. The terminal will use the wallet context from its parent tree.
</Warning>

## WalletConnect Project ID

Get a free Project ID at [cloud.walletconnect.com](https://cloud.walletconnect.com). The terminal uses it to support mobile wallet connections (MetaMask Mobile, Rainbow, etc.).

Without a valid project ID, mobile wallets will not connect — desktop browser extensions still work.
