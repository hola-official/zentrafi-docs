## Flow Overview

ZentraFi creates a continuous flow from token creation through graduation to open market trading:

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Creation   │ ──► │   Bonding    │ ──► │  Graduation  │
│              │     │   Curve      │     │              │
│  Deploy via  │     │  Trade via   │     │  Migrate to  │
│   Factory    │     │   Router     │     │ LiquidityBook│
└──────────────┘     └──────────────┘     └──────────────┘
                            │                     │
                            ▼                     ▼
                     ┌──────────────┐     ┌──────────────┐
                     │    Fees      │     │   Trading    │
                     │              │     │              │
                     │  Creator +   │     │  Zentra X    │
                     │  Treasury    │     │    (DEX)     │
                     └──────────────┘     └──────────────┘
```

## Token Lifecycle

### Phase 1: Creation

Project launches token via `LaunchpadBonding`:

```
Input:
├── Token metadata (name, symbol, decimals)
├── Social links (description, image, twitter, telegram, website)
├── Launch fee (0.01 PROS)
└── WPROS for initial purchase (optional)

Output:
├── LaunchpadToken deployed (1B supply)
├── LaunchpadPair created (virtual AMM)
└── Trading immediately available
```

### Phase 2: Bonding Curve Trading

Users trade via the virtual AMM pool:

```
User Journey:
├── Discover token on Zentra App
├── Review token details and current price
├── Buy with native PROS (auto-wrapped to WPROS)
├── Price adjusts based on reserve changes
├── Hold tokens (non-transferable until graduation)
└── Sell back to pool if desired
```

**During Trading:**
- Price follows constant product formula (x × y = k)
- Anti-sniper tax applies in first 30 seconds
- 1% buy fee, 2% sell fee
- Fees split: 20% creator, 80% treasury

### Phase 3: Graduation

When token reserve drops below 20%:

```
Graduation Sequence (Atomic):
1. Liquidity removed from LaunchpadPair
2. Token unlocked (graduated = true)
3. sZTR allocation distributed:
   └── 10% of raised WPROS to stakers
4. LiquidityBook pool created:
   ├── Token/WPROS pair
   └── Concentrated liquidity around graduation price
5. Trading transitions to Zentra X
```

**Critical:** All steps execute atomically. No manipulation window.

### Phase 4: Open Trading

Full market on Zentra X (LiquidityBook):

```
Market State:
├── Active LiquidityBook pool
├── Concentrated liquidity from graduation
├── Price determined by bin mechanics
├── LPs can add/remove positions
├── Dynamic fees based on volatility
└── Full ERC-20 transferability
```

## Liquidity Mechanics

### Pre-Graduation (Bonding Curve)

Virtual liquidity AMM:

| Component | Value | Description |
|-----------|-------|-------------|
| Token Reserve | 1B → 200M | Decreases as users buy |
| Virtual Quote | 200 WPROS | Calculated from K and assetRate |
| Real Quote | 0 → 800 WPROS | Increases as users buy |
| Total Quote | 200 → 1000 WPROS | Virtual + Real |

**Price calculation:**
$$
\text{Price} = \frac{\text{Quote Reserve}}{\text{Token Reserve}}
$$

### Graduation Transition

When token reserve hits 20%:

| Asset | Amount | Destination |
|-------|--------|-------------|
| Tokens | ~200M remaining | LiquidityBook pool |
| WPROS | 10% of raised | sZTR distributor |
| WPROS | 90% of raised | LiquidityBook pool |

**Example (default config):**
- Raised: 800 WPROS
- sZTR holders: 80 WPROS
- LB liquidity: 720 WPROS + 200M tokens

### Post-Graduation (LiquidityBook)

Concentrated liquidity in bins:

```
Bins:    -3   -2   -1    0   +1   +2   +3
         ▓▓   ▓▓▓  ▓▓▓▓████▓▓▓▓  ▓▓▓   ▓▓
                       ↑
                  Active Bin
```

Each bin provides zero-slippage trades at its price until depleted.

## Trading Flow

### Pre-Graduation: Buy Tokens

```solidity
// User buys with native PROS
uint256 tokens = bonding.buyWithNative{value: amount}(
    token,      // Token address
    minTokens,  // Slippage protection
    deadline    // Transaction deadline
);
```

**Flow:**
1. PROS auto-wrapped to WPROS
2. Anti-sniper tax applied (if within 30s)
3. Buy fee (1%) deducted
4. Remaining WPROS swapped for tokens via AMM
5. Tokens credited to user (held in contract)

### Pre-Graduation: Sell Tokens

```solidity
// Approve bonding contract first
IERC20(token).approve(address(bonding), amount);

// Sell for native PROS
uint256 pros = bonding.sellForNative(
    token,      // Token address
    amount,     // Tokens to sell
    minPros,    // Minimum PROS to receive
    deadline    // Transaction deadline
);
```

**Flow:**
1. Tokens transferred from user
2. AMM calculates WPROS output
3. Sell fee (2%) deducted
4. WPROS unwrapped to PROS
5. PROS sent to user

### Post-Graduation: Swap on Zentra X

Standard LiquidityBook trading via LBRouter:

```
1. User: Swap WPROS for TOKEN
2. Router: Find path through bins
3. Execution:
   ├── Bin 0: Provides X tokens at price P₀
   ├── Bin 1: Provides Y tokens at price P₁
   └── Continue until filled
4. Settlement:
   ├── User receives tokens
   ├── Pool receives WPROS
   └── Fees to LPs
```

## Participant Journeys

### Trader Journey

```
Goal: Buy newly launched token

Pre-Graduation:
1. Find token on Zentra App
2. Connect wallet
3. Enter PROS amount
4. Review price and tokens to receive
5. Confirm (wait 30s if avoiding anti-sniper)
6. Receive tokens (held until graduation)

Post-Graduation:
1. Navigate to Zentra X
2. Find graduated token
3. Swap WPROS ↔ TOKEN
4. Full ERC-20 control
```

### LP Journey (Post-Graduation)

```
Goal: Earn fees by providing liquidity

1. Navigate to Pools on Zentra X
2. Find graduated token pair
3. Choose liquidity shape:
   ├── Spot: Wide range, lower risk
   ├── Curve: Concentrated, higher fees
   └── Bid-Ask: Market making strategy
4. Set bin range
5. Deposit tokens
6. Monitor active bin status
7. Rebalance as price moves
8. Claim fees
```

### Project Journey

```
Goal: Launch token with automatic market

1. Configure token on Zentra App:
   ├── Name, symbol, decimals
   ├── Description and image
   └── Social links
2. Pay launch fee (0.01 PROS)
3. Token + pool deploy instantly
4. Promote to community
5. Monitor trading activity
6. Receive creator fees (20% of trades)
7. Graduation triggers automatically
8. Trading continues on Zentra X
```

## Integration Points

### For Developers

| Contract | Purpose | Key Functions |
|----------|---------|---------------|
| `LaunchpadBonding` | Launch & trade | `launch()`, `buyWithNative()`, `sellForNative()` |
| `LaunchpadFactory` | Token creation | `createToken()` |
| `LaunchpadRouter` | Swap routing | `swapExactTokensForTokens()` |
| `LaunchpadPair` | AMM pool | `getReserves()`, `swap()` |
| `LaunchpadToken` | ERC-20 token | Standard ERC-20 + `graduated()` |

### Querying Token State

```solidity
// Get current price and reserves
(uint256 tokenReserve, uint256 quoteReserve) = pair.getReserves();
uint256 price = quoteReserve * 1e18 / tokenReserve;

// Check graduation eligibility
(bool eligible, uint256 remaining, uint256 threshold) =
    bonding.isEligibleForGraduation(token);

// Check if graduated
bool graduated = LaunchpadToken(token).graduated();
```
