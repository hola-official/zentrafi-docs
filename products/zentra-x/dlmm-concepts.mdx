## Concentrated Liquidity: The Technical Advantage

DLMM enables LPs to provide liquidity within specific price ranges instead of across all possible prices.

**Why this matters in practice:**

| Pair Type | Realistic Range | Capital Utilization |
|-----------|-----------------|---------------------|
| USDC/USDT | $0.999 - $1.001 | ~100% in 3 bins |
| ETH/USDC | ±10% from current | ~80% in active region |
| New token | ±50% from launch | ~60% in active region |

Liquidity outside the trading range generates zero fees. Concentration = efficiency.

## Bin Structure Deep Dive

### Bin Pricing Formula

Each bin has a fixed price. Consecutive bins are spaced by the bin step (in basis points):

$$
P_{i+1} = P_i \times (1 + s)
$$

Where:
- $P_i$ = Price of bin $i$
- $s$ = Bin step (e.g., 0.0025 for 25 bp)

**Worked Example:**

ETH/USDC at $20.00 with 25 bp (0.25%) bin step:

| Bin ID | Calculation | Price |
|--------|-------------|-------|
| Active (0) | Base | $20.00 |
| +1 | $20.00 × 1.0025 | $20.05 |
| +2 | $20.05 × 1.0025 | $20.10 |
| -1 | $20.00 ÷ 1.0025 | $19.95 |
| -2 | $19.95 ÷ 1.0025 | $19.90 |

### Constant Sum Within Bins

Unlike constant product AMMs ($x \times y = k$), each bin uses **constant sum**:

$$
P \cdot x + y = L
$$

Where:
- $x$ = Token X reserves in bin
- $y$ = Token Y reserves in bin
- $L$ = Total liquidity in bin
- $P$ = Bin price (Δy/Δx)

**What this means:** Trades within a bin execute at exactly the bin price. No slippage until the bin depletes.

### Composition Factor

The composition factor $c$ describes what percentage of a bin is token Y:

$$
c = \frac{y}{L}
$$

From this, we can derive reserves:

$$
y = c \cdot L
$$

$$
x = \frac{L}{P} \cdot (1 - c)
$$

**Practical interpretation:**

| c value | Bin State | Meaning |
|---------|-----------|---------|
| c = 1 | 100% Y | All token X sold, bin exhausted |
| c = 0.5 | 50/50 | This is the active bin |
| c = 0 | 100% X | All token Y sold, bin exhausted |

## Market Aggregation

### The Active Bin

Only one bin is "active" at any time. The active bin:
- Contains both token types
- Is where trading happens
- Earns all trading fees

```
Price Direction →

Bins:  [Y only] [Y only] [X+Y] [X only] [X only]
                          ↑
                    Active Bin
                    (earns fees)
```

### How Price Moves

When trading exhausts the active bin, price moves to the next bin:

```
Scenario: Strong buy pressure on token X

Before:
[Y][Y][X+Y][X][X]
       ↑ Active at $20.00

After buying depletes $20.00 bin:
[Y][Y][Y][X+Y][X]
          ↑ Active at $20.05
```

Each bin flip is a discrete price change equal to one bin step.

### Multi-Bin Trades

Large trades cross multiple bins. The effective price is the volume-weighted average:

```
Trade: Buy 1000 ETH

Bin $20.00: 400 ETH available → 400 at $20.00 = $8,000
Bin $20.05: 400 ETH available → 400 at $20.05 = $8,020
Bin $20.10: 200 ETH available → 200 at $20.10 = $4,020

Total: 1000 ETH for $20,040
Effective price: $20.04
```

The slippage is discrete and predictable, not continuous.

## Fee Mechanics

### Dynamic Fee Formula

$$
\text{Total Fee} = \text{Base Fee} + \text{Variable Fee}
$$

**Base Fee:**

$$
f_b = B \times s
$$

Where $B$ is the base factor and $s$ is bin step.

**Variable Fee:**

$$
f_v = A \times (v_a)^2
$$

Where:
- $A$ = Variable fee control parameter
- $v_a$ = Volatility accumulator

### Volatility Accumulator

The volatility accumulator tracks recent market activity:

| Event | Effect on $v_a$ |
|-------|-----------------|
| Bin crossing | Increases |
| High swap frequency | Increases |
| Large trade size | Increases |
| Time passing | Decays |

**Result:** Fees automatically surge during volatile periods, protecting LPs.

## Pool Identification

Pools are uniquely identified by their tuple:

```
(Token X, Token Y, Bin Step S)
```

For example:
- `(ETH, USDC, 25)` = ETH/USDC with 25 bp step
- `(USDC, USDT, 1)` = Stablecoin pair with 1 bp step

Multiple pools can exist for the same pair with different bin steps.

## Bin ID System

Bins are identified by integer IDs relative to a reference point:

$$
\text{Bin ID} = \text{floor}\left(\frac{\log(P)}{\log(1 + s)}\right)
$$

Where:
- $P$ = Price at bin
- $s$ = Bin step

**Example:** For 25 bp step at $1.00:
- Bin ID 8388608 = $1.00 (reference)
- Bin ID 8388609 = $1.0025
- Bin ID 8388607 = $0.9975

### Calculating Expected Output

```solidity
// Pseudocode for swap calculation
function getAmountOut(uint256 amountIn, uint24 activeId) {
    uint256 remaining = amountIn;
    uint256 amountOut = 0;
    uint24 currentBin = activeId;

    while (remaining > 0) {
        (uint256 binReserveX, uint256 binReserveY) = getBinReserves(currentBin);
        uint256 price = getPriceFromId(currentBin);

        uint256 maxOutput = swapForY ? binReserveY : binReserveX;
        uint256 usedAmount = min(remaining, maxAmountForBin);

        amountOut += calculateOutput(usedAmount, price);
        remaining -= usedAmount;

        if (binDepleted) currentBin = nextBin;
    }

    return amountOut;
}
```
